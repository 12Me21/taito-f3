The F3 does not use −5V. (it is completely unconnected, i have tested this)

button5 (for 1p only) was connected on the single-board F3 only

button4 (1p and 2p) is connected, but also not used by any games afaik

note that button4 and button5 are not the same as kick1 and kick2

there are 3 service switches (on the jamma connector, 3p connector, and 4p connector) and 2 test switches (jamma connector, and the switch on the board). these can all be read separately.

Port SI: Serial Interface (MC68681 DUART)

123456
VCC TxDB RxDB TxDA RxDA GND

This connects directly to the DUART, which is accessible to the audio cpu. The default audio program outputs some text over TxDB. The clock inputs for channels A and B are 0.5 MHz and 1 MHz, respectively.

Port Z: Serial Output

1234
VCC GND data latch

 This is a serial output port for the main CPU.  It is controlled through the FIO, with bits 0 and 1 at address 0x4A0013.  All games use the same protocol for outputting data over this port.  When pin 4 goes low, a bit can be read from pin 3 (high = 1).  For each byte, 10 bits are transmtted: a start bit (0), then the data bits (lsb first), then a stop bit (1).  All values (including string characters!) are sent as BCD (2 decimal digits per byte) unless otherwise noted. e.g. "A" (ascii 65) is sent as the byte 0x65, not 0x41.  Games log events such as inserting credits, getting a game over, etc.  The functions for writing to the Z port are at the very beginning of the ROM (starting at address 0x402) and are identical across games.

Jumper JP3: joystick input mode

joy­stick2sen­sor
VCC (to resist­ors) GND

this configures the joystick inputs to use either pull-up resistors (for buttons or joysticks) or pull-down resistors (for rotary encoder dials).
