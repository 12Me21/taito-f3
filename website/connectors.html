VCC nc GND

Port G: JAMMA

The F3 does not need −5V (the pin is completely unconnected, i have tested this)

button5 (for 1p only) was connected on the single board F3 only

button4 (1p and 2p) is connected, but also not used by any games afaik

note that button4 and button5 are not the same as kick1 and kick2

there are 3 service switches (on the jamma connector, 3p connector, and 4p connector) and 2 test switches (jamma connector, and the switch on the board). these can all be read separately.

Port SI: Serial Interface (MC68681 DUART)

This connects directly to the DUART, which is accessible to the audio cpu. The default audio program outputs some text over TxDB.
The clock inputs for channels A and B are 0.5 MHz and 1 MHz, respectively

123456
VCC TxDB RxDB TxDA RxDA GND

Port Z: Serial Output

This is a serial output port for the main CPU. It is controlled through the FIO, with bits 0 and 1 at address 0x4A0013.

When pin 4 goes low (i think), a bit can be read from pin 3. Each symbol consists of 10 bits: 2 control bits and 8 data bits. (i can't remember if the output is inverted)

All games use the same protocol for outputting data over this port. They log events such as inserting credits, getting a game over, etc.

1234
VCC GND data latch
