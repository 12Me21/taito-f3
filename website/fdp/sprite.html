Sprites

The fdp renders up to ~900 sprites per frame to a single bitmap framebuffer.  Each sprite uses a 16×16 texture from rom, with up to 63 colors (plus transparent).

Sprites are split into four "groups", depending on the 2 uppermost bits of their palette.  Sprites in higher group numbers render in front of lower ones.  Within a group, later sprites render above earlier ones. i think? The Sprite Layer

The sprite framebuffer is used as the source for the Sprite Layer.

Some of the sprite layer's common layer settings can be set per-group (Priority, Blend Mode, and Alpha Select).  In some ways, it acts like 4 separate sprite layers, but note that sprites can never blend with each other because they share a single framebuffer.

Sprite Rendering

Sprites are rendered from Sprite Ram, which contains 4 banks of 1024 entries each.  Each entry consists of eight 16-bit values, representing one sprite (and/or a command that sets graphics parameters).  On each frame, the fdp iterates over the entries in the current bank (which can be changed by a command sprite).

Individual sprites can be tiled into larger blocks by using the Block Controls.

There are two scroll values, which we call "global" and "subglobal".  These can be changed at any time, and you can choose which ones (global only, global+subglobal, or neither) to use for each sprite.
