Shadow

Shadow mode takes effect when there are two layers with neighboring priority values. (within the halfpixel column)explain this better, maybe we need to define the pipeline elsewhere and then we can just say this stuff.  Anywhere both layers overlap (have nontransparent pixels), the upper layer's color is altered.  The effect varies based on the bpp setting of the upper layer's pixel.  Generally: in shadow mode 2, it sets the lowest bit in the palette index, and in mode 3, it disables the highest bit in the texture.

Shadow only works between certain combinations of upper/lower layers, though all layers can affect the results via priority conflicts (e.g. normally sprites can't shadow onto tilemaps, but you can shadow a tilemap onto another tilemap, and use conflicting sprites to cut a hole in the lower tilemap, which will cut a hole in the shadow effect)  There isn't any clear rule for which layers are compatible:
