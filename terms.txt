terms:
"layer" - one of the eight graphics layers that can contribute to the color of a pixel.
 - background color
 - pivot (text / pixel)
 - sprites
 - 4 tilemaps
 - highcolor layer

"tilemap" / "tilemap layer n" ="playfield"
"clip plane" / "clip interval" / "clip window" - one of the four clipping intervals. each interval has a "left" and "right" value
"sprite group" ="sprite priority group" - sprites are split into 4 groups based on the upper 2 bits of their palette (00,40,80,C0). these can have different blend settings
"mosaic" - horizontal pixelization effect
"background color [layer]" - color drawn behind everything else (used if nothing else is displayed there)

"color" - ambiguous
"color id" ="color index" - a color in the FDP - that is, a 13-bit number, which is later used as an index into color RAM.
"palette id" - the upper bits of a color index (while the lower bits are supplied by the texture). abstractly, an object's palette selects which set of 16 (or 32, or 64) colors is used for it.
"rgb color" - an actual color, seen and blended by the FDA and output to the screen.
"color ram" / "palette ram" - the map of color numbers to rgb colors

"priority" - blending priority (determines which 2 layers are chosen for blending etc) not to be confused with sprite priority, which only matters while rendering to the framebuffer, and is mapped to priority by lineram 7600
"blend mode" - which pixels a layer contributes to (left/right/both/neither)  this is like  "priority stack-up contribution"
"blend value" / "blend alpha value" - ranges from $7 (100%) to $F (0%). $B is 50%. represents an alpha value. there are 4 of these, 2 for the left half-pixel and 2 for the right half-pixel
"alpha select" ="blend select"/"blend value select" - setting which determines which of the sets of alpha values will be used for a given layer
"half-pixel" - one of two pixels that is sent to the FDA for each screen pixel
"A and B" / "left and right" - what we call each of the half-pixels. 

note: think of it this way: priority and half-pixel are the 2 axes of the priority cell array

"sprite" - ambiguous
"sprite list" - the list in graphics ram which the sprites are rendered from
"sprite [list] entry" - one item in the sprite list, consisting of 8 words

"tile" - the thing that a tilemap/pivot layer is composed of, represented by 32/16 bits in graphics ram

"texture" ="character" - the bitmap that a sprite, tilemap tile, or text cell is rendered from.

"pivot layer" - one of the graphics layers. 64x64 grid of 8x8px tiles, with textures defined in RAM.
this layer has 2 modes: "text mode" and "pixel mode", which affect how textures are mapped to the tiles.
 in "text mode", a tile's texture is chosen based on its texture id, from a set of 256 textures.
 in "pixel mode", each tile in the first 32 rows (the bottom half of the layer reuses the same data) references a distinct texture, effectively creating a 512x256 bitmap image (though the pixels are in a strange order)
 note that the palette and flip attributes function the same in either mode.
